Reports a call that passes the wrong number of arguments to a known function.

The argument count of a call site is compared against the arities the function actually accepts, resolved from the Phel standard library and from project definitions. Both fixed-arity and variadic (& rest) signatures are understood, including multi-arity functions.

Example:


;; inc takes exactly one argument
(inc 1 2)   ;; reported: Wrong number of args (2) passed to 'inc'. Expected: 1.

(inc 1)     ;; fine

What is checked

Only real function calls with a trustworthy textual argument count are inspected. To avoid false positives, the following are deliberately not checked:

  • Special forms and macros with variable argument shapes (if, when, let, fn, defn, cond, and, or, and similar).
  • PHP interop calls (php/..., .method).
  • Quoted or data forms — inside quote, var, or a ' / ` reader macro.
  • Arguments of a threading macro (->, ->>, as->, some->, …), where the macro shifts the effective argument count.
  • Heads that resolve to a local binding rather than the standard-library function of the same name.

This inspection is registered at error severity because a genuine arity mismatch is a call Phel will reject at compile time.
