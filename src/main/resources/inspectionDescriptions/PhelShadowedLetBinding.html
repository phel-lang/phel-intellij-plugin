Reports a let-like binding that shadows an outer binding of the same name.

When a binding reuses a name already introduced by an enclosing scope, the inner name hides the outer one for the rest of the form. This is legal Phel, but it is a frequent source of confusion, so it is surfaced as a weak warning.

The outer binding is found in an enclosing let-like form (let, if-let, when-let, loop, for, foreach, binding, dofor) or in the parameter vector of an enclosing function form (fn, defn, defn-, defmacro, defmacro-).

Example:


(defn f [x]
  (let [x (inc x)]   ;; reported: Binding 'x' shadows an outer binding.
    x))

Ignored names

The placeholder _ and names starting with & (rest markers) are never reported.
