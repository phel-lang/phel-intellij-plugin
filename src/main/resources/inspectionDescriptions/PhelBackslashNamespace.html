Reports use of the legacy backslash separator (\) in Phel namespaces inside (ns ...), :require, and :use clauses.

Since Phel 0.35 the canonical namespace form is dot-separated. Backslash separators are still accepted but are deprecated and emit a runtime warning when PHEL_WARN_DEPRECATIONS=1 is set.

Example — deprecated:

(ns my-app\core
  (:require phel\string :as s))

Example — preferred:

(ns my-app.core
  (:require phel.string :as s))

The quick-fix rewrites the offending symbol, replacing each \ with ..
